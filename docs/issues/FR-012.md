# FR-012: Admin Operations Dashboard

Status: MUS  
Labels: admin, operations

## Requirement

Admin dashboard manages schools, wallets, clearance requests, issues, disputes, and suspicious activity.

## User Story

As a platform admin, I want operational control, so that I can run the MVP safely.

## Proposed Solution

Give admin practical operational views for the MVP.

## Implementation Flow

Schools, wallets, clearance requests, issues, disputes, suspicious activity.

## Technical Approach

Follow the stack and data model in `docs/Project_Requirements.md` and the rules in `docs/Coding_Guidelines.md`. Use server-side validation, school-scoped permissions, audit logging where sensitive, and transactional database operations for wallet/payment work.

## Key Considerations

Keep the MVP focused on clearance requests, not full school management. Use safe product language, minimize child/parent data, and avoid presenting missing records as clearance.

## Acceptance Criteria

- [x] Admin dashboard exists.
- [x] Admin can view schools, wallets, requests, issues, disputes.
- [x] Admin can search/filter key records.
- [x] Admin actions require platform_admin.

## Notes

Admin overview, schools, clearance monitoring, disputes, and placeholder issue/wallet routes now exist. Demo admin access is enforced server-side through a `platform_admin` session cookie set by the admin login flow; production auth still needs to replace this temporary mechanism.
