# FR-002: School Directory

Status: MUS  
Labels: schools, admin

## Requirement

School directory stores manually seeded local schools with unclaimed/pending/active/suspended status.

## User Story

As an admin, I want schools listed before onboarding, so that the network feels real and local.

## Proposed Solution

Seed and manage local school profiles before onboarding.

## Implementation Flow

Admin creates/searches schools; claim flow selects unclaimed school.

## Technical Approach

Follow the stack and data model in `docs/Project_Requirements.md` and the rules in `docs/Coding_Guidelines.md`. Use server-side validation, school-scoped permissions, audit logging where sensitive, and transactional database operations for wallet/payment work.

## Key Considerations

Keep the MVP focused on clearance requests, not full school management. Use safe product language, minimize child/parent data, and avoid presenting missing records as clearance.

## Acceptance Criteria

- [ ] Schools can be seeded manually or by script.
- [ ] Statuses include unclaimed, pending, active, suspended.
- [ ] Admin can search/view schools.
- [ ] Claim flow can select unclaimed schools.
