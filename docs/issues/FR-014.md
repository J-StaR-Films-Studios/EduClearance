# FR-014: Mobile-Friendly UI

Status: MUS  
Labels: ui, mobile

## Requirement

The UI is mobile-friendly and fast for school owners using phones.

## User Story

As a proprietor or staff member, I want to use the app on my phone, so that it works during admissions.

## Proposed Solution

Make all MVP flows usable on phones.

## Implementation Flow

Responsive cards/forms, readable result states, obvious primary actions.

## Technical Approach

Follow the stack and data model in `docs/Project_Requirements.md` and the rules in `docs/Coding_Guidelines.md`. Use server-side validation, school-scoped permissions, audit logging where sensitive, and transactional database operations for wallet/payment work.

## Key Considerations

Keep the MVP focused on clearance requests, not full school management. Use safe product language, minimize child/parent data, and avoid presenting missing records as clearance.

## Acceptance Criteria

- [ ] Landing page works on mobile.
- [ ] Dashboard is readable on mobile.
- [ ] Forms are usable on mobile.
- [ ] Result/disclaimer text is visible on mobile.
