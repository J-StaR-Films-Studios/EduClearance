# FR-009: Free Issue Reporting

Status: MUS  
Labels: issues, contribution

## Requirement

Schools can report unresolved clearance issues for free, with structured fields and professional wording.

## User Story

As a previous school, I want to report unresolved obligations, so that other schools are warned.

## Proposed Solution

Allow active schools to report unresolved issues without wallet charge.

## Implementation Flow

Structured issue form with amount, type, session, term, note, evidence.

## Technical Approach

Follow the stack and data model in `docs/Project_Requirements.md` and the rules in `docs/Coding_Guidelines.md`. Use server-side validation, school-scoped permissions, audit logging where sensitive, and transactional database operations for wallet/payment work.

## Key Considerations

Keep the MVP focused on clearance requests, not full school management. Use safe product language, minimize child/parent data, and avoid presenting missing records as clearance.

## Acceptance Criteria

- [ ] Active schools can report issue for free.
- [ ] Required fields are validated.
- [ ] Issue status starts unresolved.
- [ ] Issue creation is audit logged.
