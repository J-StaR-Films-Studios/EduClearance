# FR-013: Audit Logs and Privacy Controls

Status: MUS  
Labels: privacy, compliance

## Requirement

Audit logs and privacy controls record sensitive actions and enforce data-minimizing, school-only access.

## User Story

As the platform owner, I want accountability, so that the product is safer and more defensible.

## Proposed Solution

Record sensitive actions and enforce minimum necessary access.

## Implementation Flow

Audit helper, server-side access rules, private/noindex areas.

## Technical Approach

Follow the stack and data model in `docs/Project_Requirements.md` and the rules in `docs/Coding_Guidelines.md`. Use server-side validation, school-scoped permissions, audit logging where sensitive, and transactional database operations for wallet/payment work.

## Key Considerations

Keep the MVP focused on clearance requests, not full school management. Use safe product language, minimize child/parent data, and avoid presenting missing records as clearance.

## Acceptance Criteria

- [ ] Sensitive actions are audit logged.
- [ ] Private routes are not publicly indexed.
- [ ] School users cannot view unrelated full records.
- [ ] Dispute/correction path exists or is stubbed.
