# FR-008: Previous School Notification

Status: MUS  
Labels: notifications, clearance

## Requirement

Previous school notification/contact flow provides dashboard notification and generated WhatsApp/manual message.

## User Story

As a school user, I want the previous school contacted, so that verification can continue outside a one-time search.

## Proposed Solution

Notify or guide contact with the previous school.

## Implementation Flow

Create dashboard notification and generated WhatsApp/manual message.

## Technical Approach

Follow the stack and data model in `docs/Project_Requirements.md` and the rules in `docs/Coding_Guidelines.md`. Use server-side validation, school-scoped permissions, audit logging where sensitive, and transactional database operations for wallet/payment work.

## Key Considerations

Keep the MVP focused on clearance requests, not full school management. Use safe product language, minimize child/parent data, and avoid presenting missing records as clearance.

## Acceptance Criteria

- [x] Previous school contact appears when known.
- [x] Active previous school receives dashboard notification.
- [x] WhatsApp/manual message can be generated.
- [x] Notification status is tracked.
