# FR-003: School Claim Registration

Status: MUS  
Labels: onboarding, auth

## Requirement

School claim and registration flow lets a school request access to an existing profile or create a new claim.

## User Story

As a school owner, I want to claim my school, so that I can use the platform officially.

## Proposed Solution

Let school owners request access to a listed or new school profile.

## Implementation Flow

Submit claim, create pending user/school, show awaiting approval.

## Technical Approach

Follow the stack and data model in `docs/Project_Requirements.md` and the rules in `docs/Coding_Guidelines.md`. Use server-side validation, school-scoped permissions, audit logging where sensitive, and transactional database operations for wallet/payment work.

## Key Considerations

Keep the MVP focused on clearance requests, not full school management. Use safe product language, minimize child/parent data, and avoid presenting missing records as clearance.

## Acceptance Criteria

- [ ] User can submit claim for existing school.
- [ ] User can submit new school request.
- [ ] Pending claim does not expose sensitive data.
- [ ] Admin can approve/reject later.
