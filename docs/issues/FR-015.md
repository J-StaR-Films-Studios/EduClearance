Status: MUS

## Requirement

Seed/demo data supports a convincing local demo with schools, wallets, and sample clearance cases.

## User Story

As the builder, I want demo data, so that I can show the MVP quickly.

## Proposed Solution

Provide safe fake data for a convincing local demo.

## Implementation Flow

Seed schools, wallets, unresolved/resolved issues, match/no-match cases.

## Technical Approach

Follow the stack and data model in `docs/Project_Requirements.md` and the rules in `docs/Coding_Guidelines.md`. Use server-side validation, school-scoped permissions, audit logging where sensitive, and transactional database operations for wallet/payment work.

## Key Considerations

Keep the MVP focused on clearance requests, not full school management. Use safe product language, minimize child/parent data, and avoid presenting missing records as clearance.

- [ ] Seed script creates match/no-match scenarios.
- [ ] Data uses fake names and numbers.
