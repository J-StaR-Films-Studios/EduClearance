# FR-020: Parent Dispute Portal

Status: Future  
Labels: parents, future

## Requirement

Parent-facing dispute/clearance portal allows parents to view and challenge records safely.

## User Story

As a parent, I want a clear dispute path, so that inaccurate records can be corrected.

## Proposed Solution

Later allow parents to securely dispute records.

## Implementation Flow

Verified parent access, evidence upload, admin resolution.

## Technical Approach

Follow the stack and data model in `docs/Project_Requirements.md` and the rules in `docs/Coding_Guidelines.md`. Use server-side validation, school-scoped permissions, audit logging where sensitive, and transactional database operations for wallet/payment work.

## Key Considerations

Keep the MVP focused on clearance requests, not full school management. Use safe product language, minimize child/parent data, and avoid presenting missing records as clearance.
