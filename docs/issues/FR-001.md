# FR-001: Landing Page Positioning

Status: MUS  
Labels: marketing, onboarding

## Requirement

Landing page explains EduClearance, pricing, privacy framing, and calls schools to claim/register.

## User Story

As a school owner, I want to quickly understand the product, so that I know why to join.

## Proposed Solution

Build official, mobile-first positioning with pricing, privacy framing, and claim/register CTA.

## Implementation Flow

Hero, problem, how it works, pricing, trust/safety, CTA.

## Technical Approach

Follow the stack and data model in `docs/Project_Requirements.md` and the rules in `docs/Coding_Guidelines.md`. Use server-side validation, school-scoped permissions, audit logging where sensitive, and transactional database operations for wallet/payment work.

## Key Considerations

Keep the MVP focused on clearance requests, not full school management. Use safe product language, minimize child/parent data, and avoid presenting missing records as clearance.

## Acceptance Criteria

- [x] Landing page is accessible at `/`.
- [x] Pricing says ₦5,000 gives 50 checks.
- [x] Copy says private school-to-school verification.
- [x] CTA routes to claim/register flow.
